# L5 Simulation Template

## Purpose

The Simulation layer explores possible outcomes based on the current Reality, Problem, Assumptions, and Variables.

A Simulation document answers the question:

> **"If these assumptions hold, what is expected to happen?"**

Simulation represents reasoning before empirical validation.

---

# Writing Guidelines

A Simulation document should:

- describe one or more possible scenarios
- explain the reasoning behind each prediction
- reference supporting assumptions and variables
- distinguish prediction from observation
- identify expected outcomes and uncertainties

Do not record experimental results in this layer.

Observed results belong in an **L6 Validation** document.

# Checklist

Before completing an L5 Simulation document, verify:

- The objective is clearly defined.
- Inputs are identified.
- Assumptions are explicit.
- Variables are considered.
- Predictions are separated from facts.
- A validation plan exists.

---

# Summary

An L5 Simulation document captures predicted behavior before evidence is collected.

Its purpose is to make reasoning explicit, compare alternative scenarios, and establish clear expectations that can be confirmed or refuted through validation.
