# L2 Problem Template

## Purpose

The Problem layer defines the objective, challenge, or question that motivates the reasoning process.

A Problem document should clearly describe **what needs to be understood, solved, decided, or achieved**.

It should not contain solutions or assumptions.

---

# Writing Guidelines

A Problem document should:

- define a clear objective
- describe the gap between the current and desired state
- establish the scope of the problem
- explain why the problem matters
- avoid proposing solutions

If assumptions are required, create a separate **L3 Assumption** document.

If the current state must be documented, create an **L1 Reality** document.

# Checklist

Before completing an L2 Problem document, verify:

- The problem is clearly defined.
- The objective is explicit.
- Scope is well bounded.
- Constraints are identified.
- No assumptions are presented as facts.
- No solution is proposed.

---

# Related Layers

Typical relationships include:

```text
Reality

↓

Problem

↓

Assumption
```

A Problem document establishes the direction of reasoning but does not explain why the problem exists or how it should be solved.

---

# Summary

An L2 Problem document defines the question that the reasoning process intends to answer.

Its role is to provide a clear objective and scope while separating the definition of the problem from assumptions, analysis, and solutions.
