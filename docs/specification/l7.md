# L7 Semantic Layer Specification

## Purpose

The L7 Semantic Layer defines the reasoning structure used by AIR.

Rather than organizing documents by topic or hierarchy, L7 organizes knowledge according to its semantic role in the reasoning process.

Every AIR Document belongs to one primary semantic layer.

---

# Design Goals

The L7 model aims to provide:

- consistent reasoning
- explicit knowledge organization
- AI-friendly document classification
- predictable document relationships
- independent semantic representation

L7 is not a workflow.

It is a semantic classification system.

---

# Layer Overview

| Layer | Name | Purpose |
|--------|------|---------|
| L1 | Reality | Record observable facts and current state. |
| L2 | Problem | Define the problem or objective. |
| L3 | Assumption | Record assumptions, hypotheses, and beliefs. |
| L4 | Variables | Identify factors that influence outcomes. |
| L5 | Simulation | Predict possible outcomes through reasoning. |
| L6 | Validation | Verify predictions using evidence or experiments. |
| L7 | Meta | Reflect on the reasoning process itself. |

---

# Semantic Model

The layers represent different semantic roles within a reasoning process.

```text
Reality

↓

Problem

↓

Assumption

↓

Variables

↓

Simulation

↓

Validation

↓

Meta
```

The sequence illustrates a common reasoning pattern.

Documents are **not required** to reference every preceding layer, nor must they form a strict linear workflow.

---

# Layer Definitions

## L1 Reality

Reality records observable facts.

This layer should describe what is currently known without interpretation.

Examples:

- observations
- measurements
- system state
- available evidence

Reality should avoid explanations or conclusions whenever possible.

---

## L2 Problem

Problem defines what should be understood, solved, or achieved.

A problem may describe:

- an objective
- a failure
- a research question
- a design challenge
- a decision to be made

Problems should be explicit and clearly scoped.

---

## L3 Assumption

Assumption records statements believed to be true but not yet fully validated.

Examples include:

- hypotheses
- expectations
- design assumptions
- constraints believed to exist

Assumptions should distinguish uncertainty from verified knowledge.

---

## L4 Variables

Variables identify the factors that influence outcomes.

Variables may include:

- controllable variables
- observable variables
- latent variables
- external dependencies
- environmental conditions

Variables provide the basis for reasoning and prediction.

---

## L5 Simulation

Simulation predicts possible outcomes.

A simulation may consist of:

- logical reasoning
- scenario analysis
- decision trees
- probabilistic models
- algorithmic simulations

Simulation represents expected behavior prior to validation.

---

## L6 Validation

Validation compares predictions against reality.

Validation may include:

- experiments
- testing
- measurements
- benchmarks
- observations
- peer review

Validation determines whether assumptions and simulations remain valid.

---

## L7 Meta

Meta reflects on the reasoning process itself.

Examples include:

- lessons learned
- reasoning quality
- framework improvements
- assumption revisions
- process optimization

Meta improves future reasoning rather than solving the current problem.

---

# Primary Layer

Every AIR Document shall define one primary semantic layer.

Example:

```yaml
layer: L3
```

The primary layer defines the document's semantic role.

---

# Secondary Layers

Documents may optionally specify additional semantic layers.

Example:

```yaml
secondary_layers:
  - L6
  - L7
```

Secondary layers provide supporting semantic context but do not replace the primary layer.

---

# Relationships Between Layers

Although independent, layers often reference one another.

Example:

```text
Reality

↓

Problem

↓

Assumption

↓

Simulation

↓

Validation

↓

Meta
```

AIR does not require a fixed execution order.

The semantic relationship is more important than document order.

---

# Document Classification

Layer classification should answer:

> **"What role does this document play in reasoning?"**

It should not answer:

- where the document is stored
- who created it
- which project it belongs to

Semantic classification is independent of organization.

---

# Validation Rules

An AIR validator should verify that:

- exactly one primary layer exists
- the primary layer is valid
- secondary layers are valid
- duplicate semantic layers are not defined

Applications may introduce additional validation rules.

---

# Design Principles

L7 follows several principles:

- semantics over hierarchy
- explicit reasoning
- modular knowledge
- independent documents
- reusable concepts

Each layer represents a distinct reasoning responsibility.

---

# Extensibility

Future AIR versions may extend L7 with:

- additional metadata
- semantic annotations
- ontology mappings
- reasoning confidence
- dependency analysis

Extensions should preserve compatibility with existing layer definitions.

---

# Summary

The L7 Semantic Layer provides a consistent framework for organizing knowledge according to its role in reasoning rather than its presentation or storage.

By assigning each AIR Document a primary semantic layer, AIR enables humans and AI systems to interpret knowledge using the same semantic structure while remaining independent of formatting, tooling, and implementation.
